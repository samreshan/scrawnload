scrawnload

Privacy Policy

Last updated: 2026-07-09

Scrawnload is a browser extension that detects video and audio on the page you're viewing and lets you preview or download it. This page describes what data the extension accesses and what it does with it.

What Scrawnload accesses

To find media on the page you're currently viewing, Scrawnload reads:

  • The URLs of <video>/<audio> elements, media-related links, and meta tags on the active tab
  • Network response metadata (URL and Content-Type header) for requests made by the active tab, to detect media that isn't directly visible in the page's HTML

What Scrawnload does not do

  • It does not send any of the above to any server. There is no backend — everything happens locally in your browser.
  • It does not track your browsing history across sites or sessions.
  • It does not collect analytics, telemetry, or usage statistics of any kind.
  • It does not sell, rent, or share any data with third parties, because no data ever leaves your device.
  • It does not require an account, login, or any form of authentication.

Where detected data lives

Detected media URLs are held in chrome.storage.session, which is in-memory, scoped to your browser session, and automatically cleared when you close your browser. The only thing the extension writes to persistent disk storage is a single boolean flag recording that you've seen the one-line "only download media you have the rights to" notice in the popup, so it doesn't show again — nothing else is retained. Nothing else is written to disk by the extension except the media file you explicitly choose to download.

Downloads

When you click Download, Scrawnload uses Chrome's built-in downloads API to save the file you selected to your computer, the same as clicking a download link normally would. For HLS streams, video segments are fetched and merged into a single file entirely inside your browser (using ffmpeg compiled to WebAssembly, running in an offscreen page) — this processing never leaves your machine either.

Permissions

Scrawnload requests the following permissions, each used only for the purpose below:

  • Read and change data on all websites you visit (host permissions) — required to detect media on whatever page you're currently viewing; Scrawnload only acts on the active tab.
  • Downloads — to save the file you chose to your device.
  • Storage — to hold the list of detected media for the current tab in memory, and the single first-run acknowledgment flag (see above).
  • webRequest — read-only observation of response URLs/headers to detect media; Scrawnload never modifies or blocks any request.
  • webNavigation — to clear the detected-media list when you navigate to a new page.
  • Tabs — to know which tab's media to show in the popup.
  • Offscreen — to run the HLS-merging engine (ffmpeg.wasm), which needs a document context the background service worker doesn't have.

Changes to this policy

If this policy changes, the "Last updated" date above will change accordingly. Material changes will also be reflected in the extension's Chrome Web Store listing.
